1. Bubble sort
алгоритм, суть которгого заклдючается в том, чтобы
с помощью двух циклов перебрать каждый элемент и сравнить его с каждым последующим.
Bubble sort имеет квадратичную сложность O(n^2), что означает, что время выполнения
увеличивается в квадрате с увеличением количества элементов
для сортировки. В связи с этим, обычно используется для сортировки небольших массивов или списка.

2.  Insertion sort
Insertion sort работает путем вставки каждого элемента в правильную
позицию в уже отсортированной последовательности. Он имеет также квадратичную сложность O(n^2),
но по сравнению с Bubble sort он имеет более быстрое время выполнения при сортировке небольших массивов,
так как обычно используется метод вставки элементов в отсортированный массив.

Итого:
Для сортировки небольших массивов я бы использовал  Insertion sort, поскольку в среднем у него лучше
скорость и эфективность(быстрее сортирует без ущерба для качества). При этом, если работать с уже почти
полностью отсортированным массивом, то можно использовать и Bubble sort
